File Structure: -Folder -File -Description
  • 'ProblemName'
    • 'ProblemName'Oracle.jar
      • This is just the .jar file that corresponds to 'ProblemName'Oracle.java
    • 'ProblemName'bug1.jar
      • This is just the .jar file that corresponds to 'ProblemName'bug1.java
    • 'ProblemName'bug2.jar
      • This is just the .jar file that corresponds to 'ProblemName'bug2.java
    • 'ProblemName'bug3.jar
      • This is just the .jar file that corresponds to 'ProblemName'bug3.java
    • 'ProblemName'.jar
      • This is just the .jar file that corresponds to 'ProblemName'.java
    • 'ProblemName'.txt
      • This is a text file containing the same text as 'ProblemName'.jar. The file is used for showing the problem code during competitions.
    • 'ProblemName'Req.txt
      • This file contains the requirements for the problem.
    • 'ProblemName'Ex.txt
      • This file contains the tooltip text for test input and output. It is seen when hovering over the test input and output fields.
    • bugList.txt
      • This file contains a description list of each of the bugs for the problem.
    • difficulty.txt
      • This file contains the difficulty value for the problem.
        (0=Very Easy, 1=Easy, 2=Medium, 3=Hard, 4=Very Hard)
    • src
      • 'ProblemName'Oracle.java
        • This is the master source code file that contains no bugs.
      • 'ProblemName'bug1.java
        • This is the source code file containing bug 1 for the problem.
      • 'ProblemName'bug2.java
        • This is the source code file containing bug 2 for the problem.
      • 'ProblemName'bug3.java
        • This is the source code file containing bug 3 for the problem.
      • 'ProblemName'.java
        • This is the source code file containing all of the bugs for the problem.
